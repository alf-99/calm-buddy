title:
In the middle of a panic attack, the world doesn't just feel loud—it feels impossible to navigate. This is Calm Buddy: a secure, multi-agent sanctuary designed to provide immediate, clinical support in a digital age where safety is often an afterthought.

I’m Anika Choudhury, and for my Google Kaggle AI Agent Capstone, I set out to solve a critical gap in digital mental health: the space where technology meets human vulnerability.



1:
The problem is stark. When a crisis hits, users have zero cognitive capacity to navigate complex medical menus or read blocks of text. They are currently stranded between 'Medical Portals' that are too slow and 'Standard AI' that is prone to hallucinations and lacks the clinical safety guardrails necessary for life-or-death moments. Unsafe AI isn't just unhelpful; it's dangerous.

2:
Calm Buddy is the solution. It’s an intelligent routing orchestrator built on Google ADK 2.0 that provides empathetic, CBT-based therapy while strictly enforcing clinical safety. It offers three things: Immediate Triage, Specialized Interventions, and Uncompromising Safety.

3:
Why agents? Because mental health is too sensitive for the 'stochastic' or unpredictable nature of standard LLMs. By using a multi-agent architecture, I isolate instructions. My specialized agents—like the Grounding and Breathing coaches—can only execute verified, evidence-based protocols. I replace 'maybe' with deterministic clinical logic.

4:
The ecosystem is powered by a Local Model Context Protocol (MCP) Server. This is unique: by serving tools like breathing patterns or emergency contacts locally, I can update clinical resources on the fly without ever risking the core security of the agent logic.


5:
Safety is the first priority. Every user input must pass through a 4-Layer Security Vault. We scrub PII for absolute privacy, block prompt injections, proactively monitor for crisis keywords, and generate structured audit logs for real-time monitoring. The AI never sees what it shouldn't.


6:
The architecture prioritizes logic over probability. The ADK workflow graph ensures that high-priority crisis states are mathematically guaranteed to bypass standard chat and hit security protocols instantly. In a crisis, there is no room for a 'wrong turn' in the code.


7:
AI autopilot is dangerous in a crisis, so I built a 'Hard Pause.' If a crisis is detected, the system uses the ADK RequestInput class to halt the workflow. It refuses to proceed until it verifies the 'biological reality' of the user's safety through forced human confirmation. We ensure a human is present when it matters most.

8:
Let's see the results. When a user reports 'spinning thoughts,' the Triage Agent delegates to the Grounding Agent. This triggers a local MCP tool that guides them through a '5-4-3-2-1' sensory exercise, pulling their focus away from the panic and back to the room.

9:
When the distress is physical—'I can't breathe'—the system routes to the Breathing Agent. It delivers a regulated 4-7-8 breathing pattern. This isn't just chatting; this is actively helping the user regulate their nervous system in real-time.

10:
In an acute crisis, the Security Checkpoint bypasses the entire triage system. It detects severe risk keywords and routes the user directly to the Crisis Support Agent, delivering immediate, confidential local helpline resources to the user's screen instantly.

11:
The advantage of this architecture is clear: it is a reliable therapy tool that is privacy-first, evidence-based, and safety-critical. By using fail-safe orchestration, we ensure the model can never be manipulated into causing harm.

12:
Calm Buddy proves that we can use advanced AI to prioritize human safety above all else. When the mind is spinning, it is necessary to ensure the user is always in safe, steady hands.
