It acts as an intelligent routing orchestrator that connects users to the exact grounding, breathing, or crisis support they need, based on their immediate state.

[1:00 — ARCHITECTURE]
(show assets/architecture_diagram.png)
Let's take a look at how this works under the hood. 
Every message sent by the user starts at the Security Checkpoint. This node scrubs personal identifiable information like phone numbers or emails, checks for malicious prompt injection attacks, and audits for crisis keywords.
If the input is safe, it proceeds to the Triage Agent. The Triage Agent assesses the user's symptoms:
If they are experiencing spinning worry or intrusive thoughts, it delegates to the Grounding Coach.
If they show physical panic symptoms like a racing heart, it routes to the Breathing Coach.
And if they are in acute distress, it routes to the Crisis Support Coach.
These coaches interact with a local Model Context Protocol server that retrieves specific box-breathing patterns, sensory grounding exercises, or local crisis hotlines.
Finally, the session is logged securely via the Log Session node.

[3:00 — SECURITY & MCP]
(show app/agent.py or app/mcp_server.py)
What makes Calm Buddy unique is its strict security design. The PII scrubber makes sure no sensitive personal data is sent to the LLM, and the prompt injection detector ensures the assistant cannot be jailbroken.
All of our therapeutic exercises are served by a local Model Context Protocol server, meaning we can update patterns and retrieve dynamic local emergency contacts on the fly without changing the core agent instructions.
